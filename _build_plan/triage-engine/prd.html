Rapture for Mac — Triage Engine

Rapture for Mac becomes the one triage path for every capture. The app already lands all captures — Siri-dictated iMessages plus notes from the Rapture iOS/Android apps — in its destination folder as raw .txt. Now the app itself makes sense of every capture the moment it arrives: structured Markdown with a YAML capture contract, deterministic classification into subfolders, and opt-in Reminders/Calendar handoff via EventKit. No account, no scripts, no external automation. An advanced tier (free, off by default) adds AI triage — on-device Apple models or a bring-your-own Anthropic key — and link enrichment. A per-destination raw-mode escape hatch keeps power users unblocked.

Deliberate product-commitment reversal: this feature knowingly reverses two documented commitments (“no built-in AI” and “no in-app categorizing of captures”). The replacement stance is output neutrality — the folder remains the integration surface and any AI can read the triaged Markdown. Link enrichment is only the second outbound-network capability ever (after Sparkle). Milestone 5 updates mission.md, CONTEXT.md, README, and PRIVACY coherently; nothing lands silently.

Swift 5.9+ SwiftUI menu-bar app macOS 14+ target System frameworks only (new work) GRDB + Sparkle (existing deps) XCTest 5 milestones, each releasable

What the app does

  • Triage on arrival

    Seconds after a capture lands, a clean Markdown note exists in the right subfolder — no schedule, no scripts.

  • The root folder is an inbox

    Any new .txt at the destination root — app-written, sync-delivered, or hand-dropped — gets triaged. In-flight files settle first.

  • A capture contract, raw never lost

    YAML frontmatter (captured, source, type, raw_media). If any formatting produced the body, the verbatim lives under ## Raw.

  • Links detected and typed

    Bare or URL-dominant captures become youtube-link or article-link and file into Links/; everything else is a voice-note in Notes/.

  • Real Reminders & Calendar events

    Opt-in EventKit handoff for clear “remind me…” captures and dated appointments — with proper macOS permission prompts. Never double-books.

  • Backlog catch-up

    On launch, pending captures drain oldest-first with menu-bar progress — including the pile accumulated before this shipped.

  • AI triage (advanced, off by default)

    Routes to Tasks/, Ideas/, Journal/, Links/ with smart titles and light formatting. On-device Apple models or your Anthropic key. Never blocks filing.

  • Link enrichment (advanced, off by default)

    YouTube transcripts and article text fetched into Links/Media/ artifacts; notes renamed to real titles on arrival. Best-effort, silent on failure.

  • Offline-proof external destinations

    Destination on an external SSD (e.g., an Obsidian vault)? When unmounted, captures queue and flush in order on remount. Never dropped, no shadow folders.

  • Raw-mode escape hatch

    Flip the destination to “raw .txt, no triage” and get exactly today’s behavior back for new arrivals.

Already provided by the existing codebase

  • Two capture sources → one folder (iMessage + iOS relay)
  • Atomic writes, collision walk, attachment sibling folders
  • CaptureGate serialization + pause/relocation deferral
  • Lenient-decode Settings/State stores, injected-dir testability
  • Ledger pattern (TTL + cap) ×3 to clone
  • Settings window + “iPhone App” section precedent
  • Menu-bar status, today count, last-error surfacing
  • Permission onboarding flows (FDA, Automation)
  • Output-folder machinery (default, sidecar, migrator, scaffold)
  • Sparkle auto-update + notarized release ritual
  • 279-test XCTest suite, test-host gating

Out of scope (this release)

  • External-service actions — no email, Slack, GitHub, or SaaS routing; the action surface is files + EventKit only.
  • Client tagging / per-client workspaces — downstream-agent territory; such notes file normally.
  • X/Facebook/Instagram enrichment — no reliable extractor; still detected and filed as links.
  • Attachment content extraction — attachments copied alongside as today; converting contents is a later release.
  • Audio transcription on the Mac — the iOS app owns transcription; orphan .m4a handling unchanged.
  • User-editable rules engine — deterministic behavior is fixed; no rules UI this release.
  • Bulk re-triage — triage happens once, at arrival; first-enable backlog catch-up is the sole exception.
  • In-app note browsing/search/editing — the folder is still the UI.
  • Multiple / per-type destinations — one destination per install; the escape hatch is a mode, not a second destination.
  • Cron/recurring automation — “every Monday…” notes file as notes; no recurrence created.
  • Triage analytics/telemetry — none, matching the app's posture.
  • Unchanged pre-existing exclusions — group chats, Mac App Store, cloud mode.

Milestones

1

Triage Engine Core

Captures become structured, filed Markdown the moment they land, with zero setup. Ships the out-of-box promise and drains the accumulated backlog on day one.

What gets built

  • On-arrival processing of any new .txt at the destination root; in-flight files settle; outputs never re-triaged
  • The Markdown capture contract incl. the ## Raw invariant and source detection
  • Deterministic triage: link detection, first-words titles, dated filenames, Notes/ + Links/ filing
  • Original .txt deleted only after durable .md write; attachments follow their note
  • Backlog catch-up oldest-first with menu-bar progress; triage ledger prevents double-processing
  • Triage on by default for everyone; raw-mode escape hatch; one-time updater notice; pause/relocation respected

Not in this milestone

  • EventKit handoff (M3), AI (M4), enrichment (M5)
  • Offline spool / external-volume handling (M2) — destination may be assumed available
  • Public docs story rewrite (M5) — changelog only
  • Bulk conversion on mode switch — new arrivals only

Done when: a Siri self-iMessage and a Rapture iOS note each produce a correct .md within seconds; a pending pile drains oldest-first with visible progress; raw mode restores v1.0.88 behavior; all tests pass.

2

Destination Resilience

External-volume destinations become first-class (reference case: an Obsidian vault on an external SSD), and “never drop a capture” becomes true even when the destination disappears.

What gets built

  • External-volume destinations; “volume absent” vs “folder missing” distinguished
  • Internal spool while offline, for captures from all sources
  • Menu bar + Settings: “Destination offline — N captures queued”
  • Ordered flush + triage on remount; today counts correct; relocation still works over the triaged tree

Not in this milestone

  • Multiple / per-type destinations
  • Sync-conflict resolution beyond atomic writes
  • A spool browser UI — status + count only

Done when: with the destination on an unplugged external drive, captures queue with an accurate count and nothing is written to the boot volume; replugging flushes everything in order into correctly triaged notes; all tests pass.

3

Reminders & Calendar Handoff

The capability external automation never had: proper TCC-permissioned handoff of clear reminder/appointment captures to Apple Reminders and Calendar.

What gets built

  • Two independent opt-in toggles with plain-language pre-prompts + TCC dialogs
  • Conservative deterministic detection: “remind me…” → Reminder; dated appointment → 1-hour event; past dates skipped; ambiguity = just file
  • Additive: the note always files; created items carry the full original text; embedded clauses handled
  • Target list/calendar pickers; handoff dedup ledger; ✓ Saved reply suffix for iMessage-sourced captures

Not in this milestone

  • Recurring items, invitees, locations, custom alerts
  • Editing/completing/undoing created items from the app
  • Review-before-create step
  • AI-improved detection (M4)

Done when: “remind me to change the furnace filter Wednesday at 9am” yields a Reminder with the right due date plus a filed note; a dated appointment yields a 1-hour event; re-dictation doesn't double-create; toggles off = pure filing; all tests pass.

4

AI Triage

The advanced tier: on-device or BYO-key AI classification, smart titles, light formatting, and sharper handoff detection — never blocking, never required.

What gets built

  • One AI toggle; engine auto-resolves (on-device on macOS 26+, else Anthropic key); active engine + honest privacy line in Settings
  • Secure key entry; classification into task/idea/journal/link → Tasks/, Ideas/, Journal/, Links/ with Notes/ fallback
  • Smart titles (concise imperatives, filler stripped, 3–10 words); light body formatting with ## Raw preserved
  • Sharper handoff detection behind the M3 toggles; deterministic fallback on any AI unavailability

Not in this milestone

  • Custom taxonomies, auto-tagging, summarization
  • Model pickers, temperature knobs, prompt editing
  • Batch re-classification of already-filed notes
  • Licensing/entitlement gating — everything free, opt-in

Done when: a rambling task dictation files into Tasks/ with a concise imperative title, formatted body, and verbatim ## Raw; with AI unavailable it files deterministically with no delay; both engines demonstrably work; all tests pass.

5

Link Enrichment & the New Story

Link captures become genuinely useful — transcripts and article text land next to the note — and the product's public story is rewritten honestly: capture → triage → your notes, no scripts required.

What gets built

  • Enrichment toggle (off by default, independent of AI); YouTube transcripts (best-effort) + article text extraction
  • Artifacts in Links/Media/ with frontmatter + backlink; one-time real-title rename; artifact link appended to the note
  • Dedup by video ID / normalized URL — duplicates point at the existing artifact, no re-fetch; quiet retries then silent give-up
  • Coordinated docs overhaul: README (story + Network badge), PRIVACY, mission.md + CONTEXT.md reversal, examples/ + scaffold reconciliation

Not in this milestone

  • X/Facebook/Instagram, paywalled or JS-rendered pages
  • Media downloads or thumbnails — text only
  • AI summarization of fetched content
  • Backfill-enrichment of previously filed links

Done when: a YouTube link yields a renamed note + transcript artifact; an article link yields an extract; repeats don't re-fetch; enrichment off = no outbound requests beyond Sparkle; all four docs tell the new story and their verification claims match the binary; all tests pass.
